- `src/frontend/semantic.rs`: Threads `arr_binding` (BindingId) and `arr_ty` (SemanticType) through the WhileIn AST node and its then-chains, resolving the array variable via `lookup_var` during semantic analysis.
- `src/frontend/semantic_types.rs`: Adds `arr_binding: BindingId` and `arr_ty: SemanticType` fields to both `SemanticWhileInChain` and the `WhileIn` variant of `SemanticStmt`.

## Decisions and direction

**while-in loop completes WhileLoop parity coverage.** CX-233's `lower_while_in_single` is architecturally significant — it's the first ranged iteration construct to get full IR lowering. The 5-block CFG pattern (entry → header → body → increment → exit) with counter parameters threaded as block arguments follows the same SSA discipline as the existing while-loop and for-loop lowerings. The decision to thread `arr_binding` and `arr_ty` through the semantic layer (rather than re-resolving at IR time) is consistent with how other lowering paths access binding metadata.

**Backend determinism train branch pattern continues.** The `train/backend-determinism` → submain merge follows the same workflow seen in previous train branches: accumulate related determinism/parity work, then integrate via a single merge PR. The branch was pruned after merge.
