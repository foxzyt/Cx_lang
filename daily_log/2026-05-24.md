## Decisions and direction

**Composite literal soundness hole closed.** The Pillar 2 audit identified that struct literals and array literals were not content-checked at the semantic layer — the analyzer validated struct name and array length but not field types, missing fields, extra fields, or element types. Programs with type-incorrect composite literals compiled and produced wrong output. This was classified as Category B (highest priority) and was fixed the same day it was formally identified. The fix adds proper field-by-field type unification, uses generic type parameter resolution for generic struct fields, and validates field completeness in both directions (no missing, no extra).

**Four-pillar audit establishes the 0.2 evidence base.** The synthesis document explicitly leaves the biggest decision (R1: commit to a shared semantic core, or keep two engines in manual sync) for the team. Everything else is ranked by evidence and grouped into quick-wins (lazy arena, `let x = expr`, humanized parse errors) and Tier-1 foundational work (builtin registry, binding-indexed locals, runtime.rs split). The audit does not prescribe a roadmap — it provides the evidence for one.

**The "two engines" structural finding.** The strongest throughline: 13 interpreter/JIT disagreements found in Pillar 2 are not 13 bugs — they're symptoms of one missing abstraction (no shared semantic core). The interpreter's SipHash cost (Pillar 1) is caused by discarding the BindingId the semantic phase already computes (Pillar 3). The 64KB memset (Pillar 1) is architecturally the arena coupled to scope creation (Pillar 3). Each pillar found the same structural issues from a different angle.
