## Decisions and direction

**When-lowering chose chained-decision CFG over SSA phi-based dispatch.** The implementation follows the same pattern as `lower_logical` (AND/OR short-circuit) — sequential Compare+Branch chains with fallthrough merging. This keeps the IR validator happy without introducing new instruction types, and matches how Cranelift's backend naturally handles multi-way branches. The alternative (table-dispatch or phi-based merge) would require new IR infrastructure that would only serve `when` and was not justified for 0.1.

**TBool unknown uses wire-value comparison, NOT unknown propagation.** The commit is explicit about the scope boundary: matching `unknown` in a `when` arm means comparing the scrutinee's wire value against the constant 2. Whether an unknown TBool infects arithmetic or comparison results is a separate post-0.1 language-design question. This is deliberately conservative — it lets pattern matching work without committing to propagation semantics.

**EnumVariant arms remain hard-rejected.** Rather than a generic "unsupported" error, the lowering emits a structured error naming the specific enum and variant. This is blocked on Enum IR lowering (separate work-item) and is the primary remaining post-0.1 `when` surface.
