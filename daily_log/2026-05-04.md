## Decisions and direction

**Submain → main merge finally happened.** After 37 days, the integration landed. This is not a language decision but a project-health decision — main was stale and the gap was the single largest drag on the project. Resolved.

**Struct and array lowering follow the same Alloca+PtrOffset+Store pattern.** CX-9 established this for structs, CX-16 adopted the same shape for arrays. Both composite types are represented as `IrType::Ptr` pointing to stack-allocated storage. Field/element access uses compile-time PtrOffset (known offset) or runtime PtrAdd (computed index). This is a deliberate design: one memory model, one addressing pattern, backends only need to handle pointer arithmetic.

**Void calls are a statement-level concern, not an expression concern.** CX-13 resolved a long-standing awkwardness: `lower_expr` returns a value, but void calls produce no value. The fix intercepts void calls at the statement level in `lower_stmt` before they reach `lower_expr`. Void calls in value position remain a semantic error.

**Backend roadmap reconciled to v4.0.** CX-11 formally acknowledged that Phase 6, Phase 10, and Phase 8 Round 1 are complete. Phase 11 is the active frontier. This closes a documentation gap where the backend roadmap still showed Phase 6 as "Up Next" despite having been done for weeks.
