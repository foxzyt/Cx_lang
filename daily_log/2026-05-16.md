## Landed today

**CX-218 (ed391c2, branch-local stokowski/CX-218): Resolve SemanticType::Numeric in lower_binary for arithmetic ops.** CONFIRMED. `lower_binary` was calling `lower_type(result_ty)` which returns `UnsupportedSemanticType` when `result_ty` is `SemanticType::Numeric`. Unresolved numeric expressions like `0 + 0` or `3 * 4` use Numeric as a placeholder type, causing `assert_eq(arith_expr, literal)` to always SKIP (exit 127). The fix mirrors `lower_value`'s existing logic — resolves `SemanticType::Numeric` to `ctx.target.numeric_literal_ir_type()` (I64 on 64-bit targets) in the arithmetic arm. Files changed: `src/ir/lower.rs` (+58 lines), `docs/backend/cx_jit_parity_checklist.md`. Arithmetic JIT parity moves from 8 PASS / 9 SKIP to 13 PASS / 4 SKIP. Zero PARITY_FAIL.
