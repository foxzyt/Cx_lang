Adds the `IndexOutOfBounds { pos, index, length }` variant to the error enum and its diagnostic renderer. Purely additive — no construction sites until the follow-up commit. Closes tracker #012.

**fix(runtime): emit IndexOutOfBounds at three array OOB sites (376fa54, submain) — CONFIRMED.** Authored 2026-05-25 13:22 EDT. Replaces the three `UndefinedVar` mis-routings in runtime.rs (lines 391, 739, 875) with the new IndexOutOfBounds variant. Programs that read/write past array bounds now get `array index N out of bounds for array of length M` instead of the nonsensical "variable 'index N' has not been declared" message. Three new verification matrix fixtures pin the behavior. Also introduces the `.jit_known_unsound` sidecar mechanism in diff_harness.rs so these fixtures are excluded from JIT parity (the JIT has no bounds checks yet — tracker #003). Closes trackers #002 and #032.

**refactor: single-source-of-truth builtin registry (45dee5a, submain) — CONFIRMED.** Authored 2026-05-25 14:03 EDT. New `src/frontend/builtins.rs` module serves as the canonical list of Cx builtins. All four hardcoded sites (semantic recognition, interpreter dispatch, IR validation, JIT lowering) now consult the registry. Adding a future builtin is a one-place change. This directly addresses the structural drift surface that produced several Pillar 2 disagreement findings. Closes tracker #008.

**perf(runtime): index per-frame variables by BindingId instead of name (5ce6cb2, submain) — CONFIRMED.** Authored 2026-05-25 17:10 EDT. The keystone performance fix from Pillar 1. `ScopeFrame.vars` is now `HashMap<BindingId, VarEntry, BuildIdHasher>` using a zero-cost identity hasher for the u32 binding IDs the semantic phase already produces. A secondary `by_name` index exists only for cold paths (string interpolation, bleed-back). Two binding-introduction sites that previously discarded their BindingId at runtime were fixed at the source: `ConstDecl` now carries a `binding` field, and `semantic_impls` carries per-method alias BindingIds. Measured results on Windows release build:
- arith_loop (5M iters): 5744ms → 2481ms (56.8% faster)
- nested_loops (4M iters): 2675ms → 1796ms (32.8% faster)
- fib_recursive: 6835ms → 6488ms (5.1% faster — call-bound, not lookup-bound)

Closes tracker #009.

## Decisions and direction

**Audit-driven tracker execution.** Today's work is the first concrete payoff from yesterday's deep-dive. Each commit explicitly references its tracker number and pillar origin. The sequence was deliberate: #012 (add the error variant) → #002+#032 (use it at three sites + add the JIT exclusion mechanism) → #008 (eliminate the structural drift surface) → #009 (the big performance win). This ordering resolves dependencies: the builtin registry was needed to cleanly validate builtin names without duplication before the BindingId refactor could safely add new fields to semantic types.

**BindingId becomes the runtime's primary key.** This is a meaningful architectural shift. The interpreter previously discarded the BindingId the semantic phase produced and re-hashed variable names at every access. Now the BindingId is the primary key throughout. This reduces the surface area that the eventual shared semantic core (deep-dive decision R1) must cover, because both interpreter and JIT now index variables the same way.

**jit_known_unsound sidecar pattern.** The OOB fixtures introduce a new test classification: programs where the interpreter is correct but the JIT is known-unsound (no bounds checking). Rather than blocking the fix on JIT parity, the harness counts these as SKIP. This prevents unsound JIT behavior from blocking interpreter correctness improvements.
