## Decisions and direction

**No-panic guarantee formalized.** CX-50 establishes a contract: on IR that passes `validate_module`, the JIT backend must not panic. Every error path that would otherwise trigger a Cranelift internal assertion is converted to a structured error. This is a meaningful correctness property — it means the JIT can be safely invoked on any validated program without risk of uncontrolled abort.

**Assert/assert_eq are no longer builtins at the IR level.** CX-48 removes `assert` and `assert_eq` from the builtin gate and lowers them to a two-branch-plus-Trap CFG pattern. This means they compile natively through Cranelift rather than requiring runtime intrinsic dispatch. The `IrTerminator::Trap` addition is the first IR terminator that doesn't transfer control — it halts execution.

**Parity baseline established.** CX-51 creates the first quantitative measurement of JIT-vs-interpreter agreement: 23/120 fixtures pass through the JIT identically, 97 skip (unsupported construct, exit code 127), 0 parity failures. This is the scoreboard that will track JIT coverage as more instructions are wired up. The zero parity failures means the JIT agrees with the interpreter on everything it can handle.

**Determinism guarantee documented.** CX-49 establishes and tests that the same IrModule produces the same exit code across multiple JIT compilations. This is a minimal but useful property for the 0.1 release — it means JIT behavior is predictable within a single session.

**Branch management at scale.** CX-44 and CX-45 are housekeeping — rebasing and verifying 6 open PRs. The fact that this requires dedicated branches with 200+ line audit docs reflects the growing number of concurrent in-flight branches (12+ active stokowski/* branches).
